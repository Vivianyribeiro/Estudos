Particionamento de requisitos por Cenários - > identificação de requisitos a partir de diferentes cenários de interaçoes entre usuario e sistema. -> PE  feita uma simulação de interações do usuáriocom o sistema na execução de uma tarefa do negócio e os usuarios explicam para o engenheiro de requisitos oque eles fazem e como o sistema poderia ajudálos a execultar esta tarefa.

-> cenarios pode ser entendidos como estórias que explicam como o sistema poderia ser usado. E PODEM SER IDENTIFICADOS a partir de aplicações técnicas tradicionais de elicitação de requisitos como ENTREVISTAS, QUESTIONÁRIOS ou ANALISE ETNOGRÁFICA

-> cenários expoe o conjunto de interações entre os atores e o sistema e revela as facilidades que são requeridas do sistema. 

-> CENARIOS podem ser escritos usando diferentes formas e padrões mas suas descrições devem conter as seguintes informaçoes:
	- uma descrição do estado do sistema antes de entrar no cenário (eventos que ativam o cenário e PRÉ-CONDIÇÕES)
	- O fluxo normal de eventos do cena´rio
	- o fluxo de exceções e e fluxos alternativos
	- Uma descrição do estado do sistema após a finalizaçõa do cenário (PÓS-CONDIÇÕES)

Caso de Uso - DEFINIÇÃO -> é uma descrição narrativa de uma seguência de eventos que ocorre quando um ator usa um ssitema para realizar uma tarefa. -> uma unidade coerente de funcionalidade provida por um sistema, manisfesta por uma seguencia de mensagens trocadas entre o sistema e um ou mais usuários externos , junto com as ações execultadas pelo sistema

Caso de Uso - OBJETIVO -> descrever a funcionalidades do sistema (requisitos funcionais)
-> mapear o escopo do sistema, onde explicita a fronteira do sistema
-> facilitar a comunicação com o usuário do sistema
-> gerenciar o projeto
->  o RUP o utiliza para guiar todo processo de desenvolvimento - RUP - Processo Unificado da Rational. Esse termo é um processo que foi criado pela empresa de engenharia de software, a Rational Software Corporation, com o intuito de orientar o desenvolvimento de um programa.
-> Mostram apenas o que o sistema faz, e não como.
	
Caso de Uso - representação gráfica

COMO ENCONTRAR CASOS DE USO ? - quais são os processos de negócios elementares que deveram ser suportados pelo uso do software - que funções o ator vai querer do sistema - o sistema armazena informações - que informações atores iram criar, ler , atualizar ou apagar - o sistema precisa notoficar o ator sobre mudanças no seu estado interno - existe algum evento externo que o sistema precisa saber, que ator informa o sistema desses aventos

CASO DE USO X CENÁRIOS -> casos de uso NÃO sao cenários, mas sim um conjunto de cenários associados a execução de um processo elementar de negócio
- cada cenário é uma instancia de um caso de uso
- para cada caso de uso, no muinimo um cenarioo para o fluxo normar(cenário principal) e outros para cada fluxo excepcional (secundário). 

TIPOS DE CASOS DE USO 

-> Primário (representam os processos principais ou mais comuns (vender produto)) 
-> Secundário - representam processo menos importantes ou mais raros (cadastrar produtos)
-> Opcional - representam processo que podem ser ignorados ou incluidos  em futuras versões (solicitar estoque par um novo produto)

FORMATO DE DESCRIÇÃO DE CASOS DE USO - 
- Nao existe padroes na industria ou na literatura para sua formatação
- deve se incluir informações que faclitem a comunicação entre os clientes e a equipe de desenvolvimento do sistema

FORMATO DE CASO DE USO - RESUMIDO
- descriçãso resumida em um parágrafo, geralmente do cenário de sucesso principal
- cridos na fase inicial de requisistos

FORMATO DE CASO DE USO - INTERMEDIÁRIO
- descriçaõ informal de parágrafsos. multiplos paragráfos que cobrem varios cenários
- podem ser usados na fase inicial de requisitos, caso haj ionteresse em maior detalhamento.

FORMATO DE CASO DE USO - COMPLETO (DETALHADO ou EXPANDIDO)
- Formato mais elaborado. todos os passso e variantes sçao descritos em detalhe
- durante a fase de requisitos, apenas os casos de uso mais importantes devem ser escritos nesse formato

ESTILO DE DESCRIÇÃO DE CASO DE USO - ESSENCIAL
- Descrição de um processo em termos de sua motivaçã e atividades essencias
- expressoes relativamente livres de detalhes de implementação, decisçoes de projeto, uso de tecnologias
  
ESTILO DE DESCRIÇÃO DE CASO DE USO - CONCRETO
- descriçaõ de um processi em termos de seru projeto real, comprometido com tecnologias de desenvolvimento, interfaces de entrada e saida
